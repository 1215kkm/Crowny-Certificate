오늘의 안건 — Crowny AI 자격증 출시 점검

현황: 3급 학습/시험/자격증발급/문의게시판까지 코드 완성, 빌드 통과. 그러나 실제 출시 가능 상태가 아님 — 데이터·결제·인증 연동이 비어있음.

  1. 🧪 [UX/출시차단] 학습→시험→발급 핵심 퍼널이 실데이터 없이 작동 불가. 3급 시험 문제 DB 시딩 + 자격증 종류 등록 안 됨
  2. 🛡️ [보안·리스크] 토스페이먼츠 키 미설정 + Firebase 승인도메인 미등록 → 결제·로그인 전면 불가. API 키 노출 점검
  3. 🎨 [UI] Crowny Class 토큰(보라→분홍 그라데이션) 사이트 전반 일관성, BGM mp3 누락
  4. 💡 [수익] 응시료(19K~49K) + 발급비(35K~55K) 외 지속사용·반복수익 경로 한 가지

참석: 강팀장·강디·강개발·강체크·아뱅

목표: 출시까지 남은 차단요소(blocker) 확정 + 우선순위 1개 결정. 한 회의 = 한 결정.

⚙️ 강개발

핵심 진단(코드 근거)

  • 의존 순서 강제됨: seed-questions는 examId가 exams에 먼저 존재해야 함(없으면 404). exams.certificateTypeId는 필수 FK, 채점은 certificateTypes.passingScore를 읽음. → certificateTypes → exams → questions 순서가 코드로 고정.
  • 진짜 차단점 = questions 시딩. 미시딩이면 questionMap 빈 객체 → 전원 0점·불합격 → 발급 불가.
  • 채점 규약 0-base: 클라/시드 correctAnswer 모두 0-base. 추가 문제 입력 시 1-base로 넣으면 전원 오답이니 주의.
  • 발급 게이트 이중검증: 클라(passed==true) + 서버(passed submission의 examId↔certificateTypeId 매칭). 데모하려면 합격 제출 1건 선행 필수.
  • ADMIN 미승격이면 시딩 자체 403(1순위 막힘), exams 누락 상태 seed 호출 404(2순위).

데모 가능 최소 시딩 셋(순서대로)

0. /api/admin/setup  → 관리자 계정 ADMIN 승격
1. /admin/certificate-types → 3급 1종 등록(GRADE_3, passingScore 70, 19000)
2. exams 도큐먼트 1개 생성(certificateTypeId=3급, questionCount 25, isActive)
3. POST /api/admin/seed-questions { examId } → 40문항 batch
4. /exams/[id]/take 정답 제출 → passed 합격 1건
5. /certificates 발급 신청 → 퍼널 종료 확인

4종 등록은 출시 항목이나 데모 비차단. 데모 차단점은 단 하나 = questions 시딩(전제: ADMIN승격→certType→exams).

핸드오프: 강팀장 — 차단점 1개로 수렴. 시딩 의존순서만 지키면 퍼널 즉시 작동.

🔍 강체크

키 노출은 합격선 — .env는 .gitignore에 잡혀 깃 히스토리 미추적, 추적되는 건 빈 .env.example뿐. 토스 시크릿키는 서버에서만(TOSS_SECRET_KEY). Firebase·토스 클라이언트 키는 NEXT_PUBLIC이라 번들에 박히나 원래 공개키라 정상. (단 과거 실키 커밋 이력 없음만 CEO가 확정.)

진짜 빨간불 = 미설정 실패가 조용히 깨짐 + "1개월 자동삭제"가 글자뿐.

🎨 강디

활성 스타일 #1 Crowny Class(보라 #8A38F5→분홍 #D53A6B). 가장 큰 마찰: 토큰이 활성 스타일과 어긋남 — globals.css --color-primary가 파랑(#2563eb), 홈 히어로는 indigo→blue. 코드 전체가 보라→분홍이 아니라 파랑 브랜드로 굴러감. 토큰 한 블록만 바꾸면 전 페이지 일괄 정렬.

둘째, BGM 깨짐: /audio/study-bgm.mp3 없는데 버튼 누르면 play()가 조용히 reject되고 토글만 "재생 중"으로 → 깨진 채 켜짐. 파일 채우거나 없으면 컨트롤 숨기는 fallback 필요.

단계이탈 원인누락 상태
홈히어로 색 브랜드 불일치 → 신뢰 흔들림토큰 정렬
학습BGM 깨짐 / iframe 로딩 공백BGM fallback, iframe loading/error
시험제출 후 무반응 불안submit loading, network error
발급토글 후 처리중 모름발급 loading, 성공토스트, 실패재시도

토큰 점검 1순위: globals.css @theme의 --color-primary→#8A38F5, --color-secondary→#D53A6B, 버튼 radius 10px. 2순위: 홈 히어로 그라데이션 indigo/blue → 브랜드 보라→분홍. 폰트 ≥16px은 충족 ✓.

핸드오프: globals.css 토큰 교체 + BGM onError로 컨트롤 숨김 + play 성공 후에만 토글 + iframe loading/error + 시험/발급 loading 상태.

📣 아뱅
자격증은 "한 번 따면 끝"이라 반복수익이 원천적으로 안 붙어요. 1회성 거래에 유효기간 2년 + 갱신료 하나만 얹읍시다. AI 도구는 반년마다 바뀌니 "2년 지난 자격증"은 신뢰가 떨어진다 — 다크패턴이 아니라 진짜 명분이고, 어학·IT 자격증이 이미 쓰는 방식. 합격 시 "유효기간 2년" 명시 → 만료 D-30 메일: "2급 자격이 30일 후 만료. 9,500원 갱신, 또는 29,000원으로 1급 도전." 손실 회피로 갱신 끌고 등급 사다리로 업셀.

만료 임박 → 상위등급 권유경쟁·구별짓기낮음(사다리 재활용)상위 응시료 업셀

선제 방어: 6개월처럼 짧으면 "돈 뜯기" 백래시 → 2년이 신뢰 임계선, 최초 응시 안내문에 유효기간 명시 필수. 측정은 만료 코호트 갱신율(목표 15%+)·업셀 전환율.

수익 경로: 유효기간 2년 → 만료 D-30 손실회피 메일 → 갱신료(반복) 또는 상위등급 응시료(업셀).

🎯 강팀장 마감 요약 — 텔레그램으로도 발송됨

강팀장은 회의 마감 시 아래 SUMMARY 블록을 *반드시* 채운다. 이 안의 내용이 텔레그램 메시지 본문으로 그대로 전송된다.

결정

  • 출시 차단요소 확정 — 코드가 아니라 "데이터·연동·운영" 3개가 빈 것이 문제. 빌드는 통과하나 실데이터·결제키·도메인 없이는 빈 화면.
  • 이번 회의 1순위 결정 = 결제·로그인 연동 복구(🔴). 데이터 시딩보다 먼저 — 둘 다 막히면 가입조차 안 되어 데모 자체 불가.
  • 수익 신규방향 채택: 자격증 유효기간 2년 + 갱신료(응시료 50%) — 반복수익+업셀. 단 출시 후 적용(출시 비차단).
  • 디자인: globals.css 브랜드 토큰이 파랑 → 활성 스타일(보라#8A38F5→분홍#D53A6B)과 불일치. 토큰 한 블록 교체로 전 페이지 정렬 + BGM graceful fallback.
