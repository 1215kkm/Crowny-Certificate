Step 2: Firestore 컬렉션 구조

  email, name, phone, address, role, createdAt

Step 4: 토스페이먼츠 직접 연동

삭제

  • src/lib/payment.ts — PortOne 유틸

생성

  • src/lib/toss-payments.ts — 승인/취소 API 유틸
  • src/app/api/payments/confirm/route.ts — 결제 승인
  • src/app/api/payments/cancel/route.ts — 결제 취소
  • src/app/payment/success/page.tsx — 성공 페이지
  • src/app/payment/fail/page.tsx — 실패 페이지

결제 흐름

  1. 프론트: 토스페이먼츠 결제위젯 SDK 로드
  2. 사용자: 결제 수단 선택 → 결제 요청
  3. 토스: 인증 완료 → successUrl로 리다이렉트 (paymentKey, orderId, amount)
  4. 서버: /api/payments/confirm에서 토스 승인 API 호출 (시크릿키)
  5. 서버: 승인 성공 → Firestore에 결제 정보 저장
  6. 프론트: 결제 완료 페이지 표시
